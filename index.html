HTML

  • The head element contains information about the webpage.
    • The code for the head element is <head> </head>
  • The body element represents the visible content shown to the user.
    • The code for the body element is <body> </body>
  • To better understand both HTML and CSS here is a series of videos by Kevin Powell on youtube, starting with my favorite explaining HTML as Wrapping Paper
  • When starting an index.html you must use <!DOCTYPE html/>
    • In order to get <, and /> typed out instead of being mistaken as code you need to type out &lt; and /&gt;
      • And to get & typed out you'll need to use & amp; without the space, but only to properly display the code
    • Next is setting the language of our HTML. We do this by using <html lang="en"/>
  • The HTML code to insert an image onto a page is <image src="img file name here" alt"description of an image"/>

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • in order to have CSS working with HTML you need to create a link to your style.css file using <link rel="stylesheet" href="'assets folder'">
    • if your images/assets aren't loading switch the order around
    • you can file/order your assets however you want, but the folder/directory must be named correctly and specified

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • remember when using github inorder to deploy your code, you'll need a jekyll theme
    • in order to create a jeykll theme you'll need a _config.yml file which you can create by manually adding a file in your repo. Name the file _config.yml then in the description add theme: jekyll-theme-'theme name'

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
